# Patent Workflow Figure Delivery Checklist

用于在附图生成、导出、嵌图与归档前做单独核对。

## 图稿源文件

- [ ] 每张附图均已保留 `.mmd` 逻辑源。
- [ ] 每张附图均已保留可编辑成品图源（优先 `.drawio`）。
- [ ] 图稿文件名与图号一一对应，不存在图1/图2混用或文件名漂移。

## 图稿生成策略

- [ ] 结构简单的框图或短流程图，才允许 Mermaid 直接输出最终图片。
- [ ] 对多分支、多回路、易出现异常折线的复杂流程图，已优先转为可编辑图源后再导出最终图片。
- [ ] 图稿默认采用白底黑字风格，不使用粉紫等演示风主题。
- [ ] 节点文字未出现明显重叠、超框、异常折行或难以辨识的问题。
- [ ] 连线未出现明显诡异折返、异常绕行或与阅读顺序相冲突的情况。

## 文内联动

- [ ] 附图说明中的图号定义与文件名一一对应。
- [ ] 正文中的“如图1所示/如图2所示”引用与附图说明一致。
- [ ] docx 中实际插入顺序与附图说明、正文引用顺序一致。

## 终稿嵌图

- [ ] 最终用于嵌图的 `.png` 或 `.svg` 已实际生成。
- [ ] 终稿 docx 已重新导出或替换图片，不是仅在目录中放置图片文件。
- [ ] 已检查终稿 docx 内 `word/media/` 非空。
- [ ] 若做过多次 patch/export，旧的测试图、临时图与备份图已清理。

## 目录归档

- [ ] 附图目录中保留 `.mmd + .drawio + .png/.svg` 三类工件。
- [ ] 根目录只保留最新正式 docx，不保留评价件与旧备份 docx。
- [ ] 需要保留的评价、审查、对比与过程说明文件已移入 `artifacts/`。
