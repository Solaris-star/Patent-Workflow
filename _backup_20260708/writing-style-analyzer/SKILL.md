# Writing Style Analyzer - 文档分析器

## 概述

本技能用于分析文档结构和写作风格，输出结构化规范供后续撰写使用。

**两种工作模式：**
1. **模板解析模式** - 解析模板文档，提取章节结构、写作规范、禁忌事项
2. **风格提取模式** - 分析参考文档（如专利），提取行文风格、叙述逻辑、用语习惯

## 何时使用

**使用模板解析模式，当用户：**
- "帮我解析这个模板的写作要求"
- "这个交底书模板各部分应该怎么写"
- "提取模板的章节结构"
- 上传模板文件并询问写作规范

**使用风格提取模式，当用户：**
- "分析这篇专利的写作风格"
- "学习 CN121526509A 的叙述方式"
- "提取参考文档的行文特点"
- 上传参考专利并希望模仿其风格

## 输入输出

### 输入
- 用户上传的文档文件（.docx/.md/.pdf）
- 用户指定的分析模式（template 或 style）
- 可选：特定分析维度（如"只分析背景技术部分"）

### 输出

**模板解析模式：**
```json
{
  "templateName": "专利技术交底书模板",
  "sections": [
    {
      "name": "技术领域",
      "purpose": "界定专利保护的技术边界",
      "required": ["所属技术领域", "应用场景"],
      "structure": "1-2 段连贯文字",
      "avoid": ["列表形式", "过于宽泛的描述"],
      "example": "本发明涉及 XX 技术领域，尤其涉及一种..."
    }
  ],
  "globalRules": {
    "language": "规范书面语，避免口语化",
    "formatting": "章节编号用中文数字（一、二、三）"
  }
}
```

**风格提取模式：**
```markdown
## 风格画像 - CN121526509A

### 背景技术
- 采用连贯段落叙述，不使用列表
- 先描述现状，再引用具体专利（带段落号）
- 缺陷分析用"然而""但是"转折，不用"其缺陷在于"

### 技术方案
- 先写系统权利要求，后写方法权利要求
- 模块描述用"XX 模块，用于..."
- 连接关系用"与所述 XX 模块连接"

### 具体实施方式
- 分为 2 个实施例：方法实施例 + 系统实施例
- 方法实施例按时间线叙述
- 系统实施例按架构层次叙述
```

## 工作流程

### 步骤 1：确认分析模式

询问用户：
1. 分析目的（解析模板 vs 学习风格）
2. 上传的文档类型
3. 是否需要特定维度的分析

### 步骤 2：读取文档

使用 `read` 工具读取用户上传的文档。

**注意：**
- .docx 文件：直接读取或转换为文本
- .pdf 文件：可能需要外部工具转换
- .md 文件：直接读取

### 步骤 3：LLM 分析

根据模式调用 LLM 进行分析：

**模板解析 Prompt 示例：**
```
请分析以下模板文档，提取每个章节的：
1. 章节名称
2. 写作目的
3. 必须包含的要素
4. 推荐结构
5. 应避免的事项
6. 示例段落（如有）

输出为 JSON 格式，结构如下：
{
  "templateName": "...",
  "sections": [...],
  "globalRules": {...}
}

模板内容：
[文档内容]
```

**风格提取 Prompt 示例：**
```
请分析以下专利文档的写作风格，提取：
1. 各章节的叙述方式（连贯段落 vs 列表）
2. 逻辑推进方式（问题→方案→效果）
3. 用语习惯（规范用语、连接词、过渡句）
4. 详略分配（技术方案 vs 实施方式的篇幅比）
5. 实施例组织方式（按时间线 vs 按模块）
6. 句式特征（长句/短句比例、主动/被动语态）

输出为 Markdown 格式的风格画像。

专利内容：
[文档内容]
```

### 步骤 4：输出结果

**模板解析模式：**
- 保存为 `workspace/analysis/template_rules.json`
- 向用户展示摘要

**风格提取模式：**
- 保存为 `workspace/analysis/style_profile.md`
- 向用户展示关键发现

### 步骤 5：确认与调整

询问用户：
- 提取的规范/风格是否准确
- 是否需要调整分析维度
- 是否需要补充特定部分的分析

## 使用示例

### 示例 1：解析专利交底书模板

**用户：** "帮我解析这个专利交底书模板，看看各部分应该怎么写"
**附件：** `专利技术交底书模板.docx`

**技能执行：**
1. 读取模板文件
2. 调用 LLM 解析各章节要求
3. 输出 `template_rules.json`
4. 展示摘要给用户

**输出摘要：**
```
✅ 模板解析完成

共提取 5 个章节的写作规范：

1. 技术领域 - 1-2 段连贯文字，界定技术边界
2. 背景技术 - 先描述现状再分析缺陷，避免排比句
3. 发明内容 - 包含技术问题、方案、有益效果
4. 附图说明 - 简要说明各附图内容
5. 具体实施方式 - 详细展开技术方案

详细规范已保存至：workspace/analysis/template_rules.json
```

### 示例 2：学习参考专利风格

**用户：** "分析一下 CN121526509A 的写作风格，我要模仿它的叙述方式"
**附件：** `CN121526509A.pdf`

**技能执行：**
1. 读取专利文件
2. 调用 LLM 分析风格特征
3. 输出 `style_profile.md`
4. 展示关键发现

**输出摘要：**
```
✅ 风格分析完成

CN121526509A 的核心风格特征：

1. 背景技术 - 连贯段落叙述，引用具体专利段落号
2. 技术方案 - 先系统后方法，模块描述规范
3. 实施例 - 2 个实施例（方法 + 系统）
4. 用语习惯 - "其特征在于"引导技术特征

详细风格画像已保存至：workspace/analysis/style_profile.md
```

## 注意事项

1. **文档格式** - 优先支持 .docx 和 .md，.pdf 可能需要额外处理
2. **长文档处理** - 如文档过长，可分段分析后汇总
3. **分析深度** - 默认中层解析（提取要求），可根据用户需求调整
4. **输出文件** - 统一保存到 `workspace/analysis/` 目录

## 与其他技能的协作

- **modular-writer** - 本技能输出的 template_rules.json 和 style_profile.md 可直接作为 modular-writer 的输入
- **deep-research / patent-workflow** - 检索到的专利或背景包可作为风格提取的输入
- **patent-workflow** - 作为专利撰写工作流的第一步
