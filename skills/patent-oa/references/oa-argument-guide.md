# OA 论证方法要点（oa-argument-guide）

patent-oa Step 4 的方法论细则。**本文件是技术论证参考，非法律意见**；条款理解以《专利审查指南》现行版与代理师解读为准。

## 三步法（创造性 22.3 的标准框架）

### 第一步：确定最接近的现有技术

- 核对审查员选定的 D1 是否真的「最接近」：技术领域是否相同、所解决的技术问题是否相近、公开的技术特征数量。
- 可质疑点：D1 领域不同且改造动机缺失；D1 目的与本申请相反（教导偏离，teach away）。

### 第二步：确定区别特征与实际解决的技术问题

- 区别特征 = 本申请权利要求特征 − D1 披露特征（以 feature_matrix 为准，逐项列出）。
- **实际解决的技术问题由区别特征在本申请中达到的技术效果确定**——审查员常见操作是把技术问题上位化（「如何提高效率」），使任何改进都显得显而易见；对策：把问题拉回说明书记载的**具体**效果（「如何在接管请求冲突时避免控制权震荡」），效果描述回指说明书段落。

### 第三步：判断对本领域技术人员是否显而易见

论证「结合启示不存在」的三个抓手：

1. **启示来源核查**：D2 中被引特征在 D2 里起的作用，与它在本申请中起的作用是否相同？作用不同 → 结合启示不成立。
2. **结合障碍**：D1 与 D2 的技术体系是否存在实际障碍（架构冲突、前提互斥、性能相互抵消）——障碍要具体到技术细节，不能空喊「难以结合」。
3. **预料不到的技术效果**：组合产生了超出各部分简单叠加的效果（回指说明书的效果记载；无实验数据时用机制推理，不编数据）。

## 新颖性（22.2）单独对比原则

- 逐特征与**单篇**对比文件比对；多篇拼凑不能否定新颖性（那是创造性问题）。
- 警惕审查员的「隐含公开」扩张：隐含公开要求本领域技术人员从该文件**直接地、毫无疑义地**确定该特征——凡需要推理补齐的都不是隐含公开。
- 惯用手段直接置换：注意 22.2 语境下的抵触申请只论新颖性。

## 充分公开（26.3）与支持（26.4）回应

- 26.3：逐点回指说明书段落，证明「本领域技术人员按记载能实现」——必要时引用实施例的具体步骤链。
- 26.4：权利要求概括是否超出说明书支持范围；回应策略优先「指出支持段落」，其次才是缩限修改。

## 修改策略（33 条红线）

- 一切修改内容必须能从原说明书与权利要求书**直接地、毫无疑义地**导出；每处修改标注依据段落。
- 常用手法：从属权利要求特征上提合并；说明书实施例特征补入；删除被完全披露的独权另立缩限独权。
- 禁止：引入原文没有的数值范围端点、补入原文未记载的效果描述。

## 常见审查员逻辑与应对速查

| 审查员操作 | 应对抓手 |
|---|---|
| 技术问题上位化 | 第二步拉回说明书具体效果 |
| 「常规技术手段」一笔带过 | 要求指出证据；论证该手段在本领域并非常规（有反向惯例） |
| 多篇文件机械拼凑 | 逐一核查结合启示与作用一致性 |
| 隐含公开扩张 | 「直接地、毫无疑义地」标准反诘 |
| 忽略功能性限定的结构含义 | 依说明书解释功能特征的具体实现边界 |
