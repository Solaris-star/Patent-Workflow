# 附图交付检查清单（FIGURE_DELIVERY_CHECKLIST）

用于附图生成、导出、嵌图与归档前的单独核对。

## 图稿源文件

- [ ] 每张附图均已保留 `.mmd` 源（**mmd 即是可编辑源**，不要求额外 drawio/vsdx）。
- [ ] 图稿文件名与图号一一对应，不存在图1/图2混用或文件名漂移。
- [ ] part_04 中每张图文字说明后紧跟该图可见 Mermaid 代码块。

## 图稿生成策略

- [ ] 优先用 Mermaid（`.mmd`）表达逻辑与节点关系。
- [ ] 需要嵌图时再渲染 `.png`/`.svg`；渲染失败须明示，不得静默跳过。
- [ ] 图稿采用白底黑字风格，未使用演示风主题。
- [ ] 节点文字无明显重叠、超框、异常折行（以 mmd 语义清晰为准；png 为辅助）。

## 文内联动（四方一致）

- [ ] 附图说明中的图号定义与附图文件名一一对应。
- [ ] 正文中「如图1所示/如图2所示」引用与附图说明一致。
- [ ] docx 中实际插图顺序与附图说明、正文引用顺序一致。
- [ ] facts_ledger 中 `mermaid_source_embedded_in_docx = true`。

## 终稿嵌图

- [ ] 最终嵌图 `.png`/`.svg` 已实际生成（若本轮交付含图）。
- [ ] 终稿 docx 已真实嵌入图片（非仅目录中存在图片文件）。
- [ ] 已检查终稿 docx 内 `word/media/` 非空。
- [ ] 多次 patch/export 遗留的测试图、临时图、备份图已清理。

## 目录归档

- [ ] 附图目录至少保留 `.mmd`；有嵌图时保留对应 `.png`/`.svg`，同图同前缀。
- [ ] 不强制保留 `.drawio` / `.vsdx`。
- [ ] 交付根目录只保留最新正式 docx。
- [ ] 评价、审查、对比与过程说明文件已移入 `artifacts/`。
