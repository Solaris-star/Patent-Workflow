# 附图交付检查清单（FIGURE_DELIVERY_CHECKLIST）

用于附图生成、导出、嵌图与归档前的单独核对。

## 图稿源文件

- [ ] 每张附图均已保留 `.mmd` 逻辑源。
- [ ] 每张附图均已保留可编辑成品图源（优先 `.drawio`，模型直接生成 XML）。
- [ ] 图稿文件名与图号一一对应，不存在图1/图2混用或文件名漂移。

## 图稿生成策略

- [ ] 结构简单的框图或短流程图，才允许 Mermaid 直接输出最终图片。
- [ ] 多分支、多回路、易出现异常折线的复杂流程图，已以 `.drawio` 为成品真源导出最终图片。
- [ ] 图稿采用白底黑字风格，未使用演示风主题。
- [ ] 节点文字无明显重叠、超框、异常折行。
- [ ] 连线无诡异折返、异常绕行或与阅读顺序冲突。

## 文内联动（四方一致）

- [ ] 附图说明中的图号定义与附图文件名一一对应。
- [ ] 正文中「如图1所示/如图2所示」引用与附图说明一致。
- [ ] docx 中实际插图顺序与附图说明、正文引用顺序一致。
- [ ] part_04 中每张图的文字说明后紧跟该图的可见 Mermaid 源码代码块，且 facts_ledger 中 `mermaid_source_embedded_in_docx = true`。

## 终稿嵌图

- [ ] 最终嵌图 `.png`/`.svg` 已实际生成。
- [ ] 终稿 docx 已真实嵌入图片（非仅目录中存在图片文件）。
- [ ] 已检查终稿 docx 内 `word/media/` 非空。
- [ ] 多次 patch/export 遗留的测试图、临时图、备份图已清理。

## 目录归档

- [ ] 附图目录保留 `.mmd + .drawio(/.vsdx) + .png/.svg` 三类工件，同图同前缀。
- [ ] 交付根目录只保留最新正式 docx。
- [ ] 评价、审查、对比与过程说明文件已移入 `artifacts/`。
