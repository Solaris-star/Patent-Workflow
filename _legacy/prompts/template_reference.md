# 专利交底书标准模板

## 章节结构

```
一、技术领域
（1-2句，限定技术领域）

二、背景技术
2.1 与本申请相关的现有技术背景知识
2.2 与本申请相关的最接近的现有技术
2.3 现有技术的缺陷和不足

三、发明内容
3.1 本申请所需要解决的技术问题
3.2 本申请的技术方案
3.3 本申请的技术效果

四、附图说明
（每图一组：图号 + 描述段落 + mermaid代码块，不少于4张图）

五、具体实施方式
5.1 系统架构实施例
5.2 方法流程实施例
```

## 编号规则

| 元素 | 格式 | 示例 |
|------|------|------|
| 章 | 中文数字 + 、 | 一、技术领域 |
| 背景子节 | 2.1 / 2.2 / 2.3 | 2.1 与本申请相关的现有技术背景知识 |
| 发明子节 | 3.1 / 3.2 / 3.3 | 3.1 本申请所需要解决的技术问题 |
| 实施子节 | 5.1 / 5.2 | 5.1 系统架构实施例 |
| 步骤 | S101-S10x | S101步骤，采集多模态传感器数据 |
| 图 | 图1-图N | 图1 系统架构 |

## 笔法规范

| 规则 | 要求 | 禁止 |
|------|------|------|
| 自称 | 本申请 | 本发明 |
| 排比 | 第一，第二，第三， | 其一，其二， |
| 起手句 | 下面对照附图，通过对较优实施例的描述，对本申请的具体实施方式作进一步详细说明。 | 简化版 |
| 背景过渡 | 由此可见，现有技术至少存在以下不足： | 直接列举 |
| 发明目的 | 本发明的目的在于提供……以解决现有技术中……的问题，实现…… | 省略 |
| 技术问题 | 本发明要解决的技术问题在于：…… | 省略 |

## 附图规范

- 不少于4张图
- 系统架构图：flowchart + subgraph 分层
- 方法流程图：flowchart
- 数据结构图：classDiagram
- 状态转移图：stateDiagram
- 每图格式：图号 + 图名 → 描述段落 → mermaid 代码块
- 不得使用 ASCII 文字框图

## 禁止项

- 文档头部元数据块（案件名称/技术联系人/注意事项）
- 权利要求书
- 1.1/1.2 编号（背景用 2.1-2.3）
- 六、其它 独立章节
- 实施例 独立章节（应合并进 五、具体实施方式）
- 自检清单
- 非 CN 专利号
- 英文术语
- 仓库/技能脚注
